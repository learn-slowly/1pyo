# 2026한표정의 PRD

## 1. 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | 2026한표정의 (정의당 지킴이 모집) |
| 서브 카피 | 한 표의 정의를 지키는, 정의당의 한 표 지킴이 |
| 한 줄 설명 | 정당의 투개표 참관인을 교육·모집·관리하는 웹 플랫폼 |
| 목적 | 시도당별 투개표 참관인 모집·배치를 자동화하고, 참관인 교육과 정당 정체성 각인을 동시에 달성 |
| 타겟 사용자 | ① 참관인 신청자 (당원/비당원), ② 시도당 관리자, ③ 중앙 관리자 |
| 기술 스택 | Next.js + TypeScript + Tailwind CSS |
| 데이터 | Google Sheets API |
| 배포 환경 | Vercel |
| 적용 범위 | 경남도당 우선, 전국 확장 가능한 구조 |
| 대상 선거 | 2026년 6월 3일 제9회 전국동시지방선거 |
| 상태 | 기획 단계 |

## 2. 배경 및 문제 정의

### 2.1 현재 상황

2025년 대통령선거에서 정의당 경남도당이 구글폼 + 구글시트 수동관리로 투개표 참관인을 모집·운영한 경험이 있다. 반응은 좋았으나 다음과 같은 문제가 있었다:

- **빈 투표소 찾기 어려움**: 모집 후반에 신청자들이 자리 있는 투표소를 하나하나 확인해야 해서 원성이 많았음
- **교육 부재**: "시스템 밖에서 알아서 교육하라"고 했더니 결국 안 됨
- **정체성 각인 부재**: 참관인이 자신이 어떤 정당 몫으로 가는지, 후보가 누구인지 인지하지 못하는 경우 발생
- **수작업 한계**: 선관위에서 선거구별로 쪼개서 주는 투표소 데이터를 시도당이 수작업으로 취합
- **전국 확장 불가**: 경남 전용 구조로 다른 시도당에 이전 불가

### 2.2 해결 방향

- 교육 → 퀴즈 → 신청의 일체화된 흐름
- 실시간 잔여석 표시로 빈 투표소 쉽게 찾기
- 시도당별 독립 운영 가능한 멀티테넌트 구조
- CSV 업로드 → 자동 통합 기능

## 3. 핵심 기능

### 3.1 교육 콘텐츠 + 퀴즈 시스템

- **설명**: 참관인 신청 전에 교육 콘텐츠를 보고 퀴즈를 통과해야 신청 가능
- **사용자 시나리오**:
  1. 사용자가 사이트 접속
  2. 모드에 따라 비밀번호 입력 (당원전용) 또는 바로 진입 (공개)
  3. 교육 콘텐츠 확인 (참관인 실무 + 정의당 정체성 + 후보 소개)
  4. 퀴즈 풀기 → 틀리면 교육 콘텐츠로 돌아가서 다시 보고 재도전
  5. 통과하면 신청 폼으로 이동
- **퀴즈 구성**:
  - 공통 파트 (전국 동일): 참관인 법적 권한, 금지행위, 이의제기 절차, 정의당 정체성
  - 지역 파트 (선거구별): 해당 선거구 정의당 후보 성명, 기호, 주요 공약
- **데이터**: 퀴즈 문항은 Google Sheets에서 관리 (관리자가 시트에 입력하면 사이트에 자동 반영)
- **우선순위**: 필수

### 3.2 참관인 신청 폼

- **설명**: 사전투표/본투표/개표 중 선택하여 투표소와 시간대를 골라 신청
- **사용자 시나리오**:
  1. 참관 유형 선택: 사전투표 참관 / 본투표 참관 / 개표 참관
  2. 시도 → 시군구 선택
  3. 투표소 선택 (잔여석 표시, 마감된 곳 비활성화, "빈 곳만 보기" 필터)
  4. 시간대 선택
  5. 신청자 정보 입력
  6. (공개 모드) 경고문 표시: "당원 조회 후 최종 알림 / 당원 아닐 시 무통보 삭제"
  7. 전체 모드 공통 경고문: "문의는 문자로만 받습니다. 전화 문의 시 참관인 신청 내역이 무통보 삭제됩니다."
  8. 신청 완료 → Google Sheets에 기록
- **좌석 구조**:

| 참관 유형 | 장소 단위 | 좌석 수 | 비고 |
| --- | --- | --- | --- |
| 사전투표 참관 | 읍면동 사전투표소 | 4석 (1일차 오전/오후 + 2일차 오전/오후) | 사전투표소 CSV 별도 |
| 본투표 참관 | 선거구별 투표소 | 2석 (오전/오후) | 본투표소 CSV 별도 |
| 개표 참관 | 시군구 개표소 | 6석 | 개표소 CSV 별도 |

- **중복 신청 방지**: 연락처 기준 동일 투표소 중복 불가
- **추첨 모드**: 관리자 토글로 시도당 전체 ON/OFF. 정원 찬 투표소도 추첨 신청 추가 접수. 신청자에게 "추첨 신청입니다" 안내 표시
- **우선순위**: 필수

### 3.3 내 신청 확인 페이지

- **설명**: 이름 + 연락처로 자기 신청 내역 조회
- **사용자 시나리오**: 사용자가 이름과 연락처 입력 → 신청한 투표소, 시간대, 상태 (신청완료/확정/추첨대기) 확인
- **데이터**: Google Sheets에서 조회
- **우선순위**: 필수

### 3.4 관리자 페이지

- **설명**: 시도당 관리자가 참관인 신청 현황을 관리
- **인증 방식**: 시도당별 고유 비밀번호 (간편) + 구글 계정 로그인 (보안 강화) 둘 다 지원
- **기능 목록**:
  - 해당 시도 신청자 목록 조회/필터 (시군구별, 투표소별, 참관유형별, 시간대별)
  - 신청자 상태 관리 (신청완료 → 확인 → 배치완료)
  - CSV 업로드: 본투표소 CSV, 사전투표소 CSV, 개표소 CSV (선관위 데이터 다수 파일 업로드 시 자동 병합)
  - 추첨 모드 토글 (시도당 전체 ON/OFF)
  - 모집 모드 설정: 당원전용 (비밀번호 잠금) / 공개
  - 교육 콘텐츠 관리: 영상 링크/이미지 등록
  - 신청자 데이터 CSV 다운로드
- **우선순위**: 필수

### 3.5 중간 관리자 (모집 책임자) 모드

- **설명**: 현장에서 참관인을 모아오는 책임자가 모아온 사람들을 일괄 입력할 수 있는 전용 페이지
- **사용자 시나리오**:
  1. 시도당 관리자가 중간 관리자 계정 생성 (이름 + 연락처 + 담당 지역)
  2. 중간 관리자가 전용 페이지 로그인 (연락처 + 비밀번호)
  3. 모아온 사람들을 일괄 입력:
     - 웹 폼에서 한 줄씩 추가 (+ 버튼으로 행 추가, 이름/연락처/투표소/시간대 입력)
     - 또는 엑셀/CSV 파일 업로드로 한번에 등록
  4. 자기가 등록한 사람들의 신청 현황 조회
- **관리자 측 기능**:
  - 중간 관리자별 모집 실적 (누가 몇 명 모았는지) 현황 조회
  - 중간 관리자 계정 생성/삭제
- **데이터**: 신청자 시트에 `recruiter` 컬럼 추가 (중간 관리자 이름/ID)
- **주의**: 중간 관리자가 등록한 사람도 퀴즈 통과 여부 처리가 필요 → 중간 관리자 본인이 퀴즈를 통과한 상태에서만 일괄 등록 가능. 개별 신청자의 퀴즈는 면제 (중간 관리자가 교육 책임)
- **우선순위**: Phase 2

### 3.6 투표소 데이터 자동 병합

- **설명**: 선관위에서 선거구별로 쪼개서 제공하는 투표소 엑셀 파일들을 한꺼번에 업로드하면 하나의 통합 시트로 자동 병합
- **사용자 시나리오**: 관리자가 선거구별 엑셀 파일 여러 개를 드래그앤드롭 → 자동 병합 → 통합 데이터 확인 → 확정
- **입력**: 선관위 투표소 CSV/엑셀 파일 (본투표소, 사전투표소, 개표소 각각)
- **출력**: Google Sheets에 통합 데이터 기록
- **우선순위**: 필수

## 4. 기술 아키텍처

### 4.1 기술 스택

- **프레임워크**: Next.js 14+ (App Router) + TypeScript
- **스타일링**: Tailwind CSS
- **데이터베이스**: Google Sheets (Google Sheets API v4)
- **인증**: 시도당 비밀번호 (환경변수 또는 시트) + NextAuth.js (Google OAuth)
- **배포**: Vercel
- **파일 처리**: SheetJS (xlsx) — CSV/엑셀 파싱 및 병합

### 4.2 외부 API/데이터 소스

| API/소스 | 용도 | 키 필요 여부 | 비고 |
| --- | --- | --- | --- |
| Google Sheets API v4 | 데이터 읽기/쓰기 | 서비스 계정 키 | 신청자, 투표소, 퀴즈 데이터 |
| Google OAuth | 관리자 로그인 | OAuth 클라이언트 ID/Secret | NextAuth.js 사용 |
| 선관위 투표소 데이터 | 투표소 마스터 데이터 | 없음 | CSV/엑셀 파일 업로드 방식 |

## 5. 데이터 모델

### 5.1 Google Sheets 구성

하나의 스프레드시트에 여러 시트(탭)로 구성한다. 시도당별로 별도 스프레드시트를 사용하되, 동일한 구조를 따른다.

#### Sheet: `설정` (config)

| 컬럼 | 설명 | 예시 |
| --- | --- | --- |
| key | 설정 키 | mode |
| value | 설정 값 | members_only |

주요 설정값:

- `mode`: `members_only` (당원전용) / `public` (공개)
- `password`: 당원전용 모드 비밀번호
- `lottery_mode`: `on` / `off` (추첨 모드)
- `sido`: 시도명 (경상남도)

#### Sheet: `본투표소` (polling_stations)

| 컬럼 | 설명 | 예시 |
| --- | --- | --- |
| id | 자동 ID | PS-001 |
| sigungu | 시군구 | 진주시 |
| electorate | 선거구 | 진주시 가선거구 |
| station_name | 투표소명 | 진주초등학교 |
| address | 주소 | 진주시 중앙로 1 |
| am_count | 오전 신청 수 | 0 |
| pm_count | 오후 신청 수 | 0 |
| am_max | 오전 정원 | 1 |
| pm_max | 오후 정원 | 1 |

#### Sheet: `사전투표소` (early_stations)

| 컬럼 | 설명 | 예시 |

## 7. UI/UX 가이드

### 7.1 기본 방향

- **모바일 퍼스트**: 신청자는 대부분 모바일에서 접속
- **디자인 톤**: 깔끔하고 신뢰감 있는 느낌. 정의당 컬러 (노란색 #FFCC00) 포인트 사용
- **접근성**: 큰 버튼, 명확한 안내 텍스트. 디지털 기기에 익숙하지 않은 사람도 쉽게 사용

### 7.2 주요 화면 흐름

```
[랜딩]
  ├─ 당원전용 → [비밀번호 입력] → [교육] → [퀴즈] → [신청 폼] → [완료]
  ├─ 공개     → [교육] → [퀴즈] → [신청 폼 + 경고문] → [완료]
  ├─ [내 신청 확인] → 이름+연락처 → 결과
  └─ [관리자] → 로그인 → 대시보드
```

### 7.3 핵심 UI 요소

- **투표소 선택**: 카드 형태. 각 카드에 투표소명, 주소, 잔여석 (예: "오전 0/1 | 오후 1/1") 표시. 만석이면 회색 비활성화 + "마감" 뱃지. 추첨 모드면 "추첨 신청 가능" 뱃지
- **퀴즈**: 한 문항씩 화면에 표시. 정답/오답 즉시 피드백. 오답 시 해설 표시 후 "교육 다시 보기" 버튼
- **경고문**: 공개 모드에서 신청 폼 상단에 빨간색 배경으로 크게 표시

## 8. 교육 영상 제작 가이드

### 8.1 제작 방식

AI 음성 + 슬라이드/모션그래픽 방식으로 제작한다. 사람이 직접 출연하지 않는다.

### 8.2 구성

| 파트 2: 참관인 실무 | 참관인 역할, 권한, 금지행위, 시정 요구 방법, 투표함 봉인 절차 | 3분 |

### 8.3 제작 도구

#### AI 음성 (TTS)

- **추천**: Typecast (한국어 자연스러움 최고), CLOVA Voice, Google Cloud TTS
- **톤**: 차분하지만 단호한 톤. 뉴스 앵커 느낌보다는 선배가 알려주는 느낌
- **주의**: 무료 플랜 범위 확인. Typecast 무료는 월 제한 있음

#### 슬라이드/모션그래픽

- **추천 도구**: Canva (모션그래픽 템플릿), CapCut (편집), PowerPoint/Keynote → 녹화
- **스타일**: 정의당 컬러 (노란색 #FFCC00) 포인트, 깔끔한 인포그래픽, 큰 글씨
- **시각 자료 포함**: 실제 투표지 예시 이미지 (선관위 공개 자료), 투표소 배치도, 참관인석 위치, 투표함 봉인 과정 사진

#### 편집/합성

- **추천 도구**: CapCut (무료, AI 음성 + 슬라이드 합성 쉬움), DaVinci Resolve (무료, 고급)
- **포맷**: 가로 16:9 (유튜브/웹 재생용) + 세로 9:16 버전 (모바일 최적화)
- **호스팅**: YouTube 비공개 링크 또는 Google Drive → 사이트에 임베드

### 8.4 제작 워크플로우

1. **대본 작성** — Claude로 초안 작성 → 레고 검토/수정
2. **AI 음성 생성** — 대본을 TTS 도구에 넣어 음성 파일 생성
3. **슬라이드 제작** — Canva 또는 PPT로 각 파트별 슬라이드/인포그래픽 제작
4. **합성** — CapCut에서 음성 + 슬라이드 타이밍 맞춰 합성
5. **자막 추가** — CapCut 자동 자막 기능 활용 (접근성)
6. **업로드** — YouTube 비공개로 업로드 → 사이트에 링크 삽입

### 8.5 대본 톤앤매너

- 실무 파트: 정확하고 명확하게. "\~해야 합니다", "\~할 수 없습니다"
- 정체성 파트: 직설적이고 냉소적. "수당 받으러 오신 거 아니죠?", "정의당의 한 표를 지키러 가는 겁니다"
- 전체적으로: 짧은 문장, 반복 강조, 핵심만

### 8.6 우선순위

- Phase 1에서는 교육 콘텐츠를 텍스트+이미지로 먼저 구현
- 영상은 Phase 2에서 제작하여 교체
- 영상 없이도 퀴즈만으로 시스템은 작동함

## 9. 제약 사항 및 주의점

### 9.1 반드시 지켜야 할 규칙

- Google Sheets API 호출은 Rate Limit 주의 (100 requests/100 seconds/user). 투표소 목록은 캐싱 적용
- 신청 시 좌석 카운트 업데이트는 원자적으로 처리 (동시 신청 시 초과 방지)
- 개인정보 (이름, 연락처) 처리 주의: HTTPS 필수, 불필요한 로깅 금지
- 퀴즈 통과 없이 신청 폼 직접 접근 차단 (서버 사이드 검증)

### 9.2 하지 말아야 할 것

- 선관위 질의 결과가 나오기 전까지 후보 소개 퀴즈를 공개 모드에서 노출하지 않음 (당원전용 모드에서만 우선 적용)
- 투표소 데이터를 DB에 영구 저장하지 않음 (Google Sheets가 단일 소스)
- 신청자 데이터를 외부에 노출하지 않음

### 9.3 성능 요구사항

- 투표소 목록 로딩 3초 이내
- 신청 완료까지 5초 이내
- 동시 접속 100명 처리 가능 (Google Sheets API 한계 내)

### 9.4 법적 주의사항

- 선관위 질의서 제출 예정 (후보 소개 퀴즈의 공개 모드 노출 허용 여부)
- 질의 결과에 따라 퀴즈 구성 변경 가능성 있음
- 선거운동기간 전/중 운영 방식 차이 확인 필요
